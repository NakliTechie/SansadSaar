📘 New here? Open the full user guide → — covers AI setup, BYOK costs/limits, web search enrichment, privacy details.

What is SansadLocal?

A single-file browser app to browse, search, and summarise reports from India's 24 Departmentally Related Standing Committees (DRSCs) — 16 chaired by Lok Sabha, 8 by Rajya Sabha. The reports are scraped daily from sansad.in and mirrored as static JSON/text on GitHub Pages so this page can read them with no server, no auth, no API key.

Why committee reports matter

DRSCs are the most robust institutional mechanism through which Indian Parliament scrutinises the executive. Their reports — on demands for grants, bills, and policy subjects — are evidence-based and non-partisan. They're also poorly indexed and rarely read.

Where AI fits

Open any report and you get four tabs: Details (metadata + PDF links), Full text (when extracted), AI summary (one click → plain-English briefing), and Ask (chat with the report). AI runs on your device by default — open Settings (the gear icon, top right) to load a model or set a BYOK key. Full walkthrough in the user guide.

Filtering

  • Search — substring match against title + extracted full text (when available)
  • Committee — narrow to one of 24 DRSCs
  • Lok Sabha — current is LS18; older terms appear as the mirror back-fills them
  • Category — auto-tagged from the title: DFG Demand for Grants, AT Action Taken, BILL Bills, ASSURE Assurances, SUBJ Subjects

Status badges

  • text — full text has been extracted from the PDF and is searchable
  • summary — you've generated an AI summary and it's cached
  • LS Lok Sabha-chaired · RS Rajya Sabha-chaired

Local AI (default)

Runs Gemma 4 (E2B or E4B) entirely in your browser via WebGPU. The first load downloads weights from Hugging Face (~1.5 GB for E2B, ~4.9 GB for E4B); subsequent loads are instant from your browser cache. Requires a recent Chrome/Edge or Firefox 130+ with a compatible GPU. No data leaves your device.

BYOK (Bring Your Own Key)

If WebGPU isn't available or you want a stronger model, configure a provider in Settings. Free options:

  • Gemini — 15 req/min free at aistudio.google.com
  • Groq — fast free tier at console.groq.com
  • OpenRouter — free models at openrouter.ai
  • Ollama — runs locally on your computer; no cloud. ollama pull llama3.2 then enable CORS for browsers.

Paid: Anthropic (best summaries) at console.anthropic.com, OpenAI at platform.openai.com.

What stays in your browser

  • API keys (in localStorage, never sent anywhere except your chosen provider)
  • Generated summaries (cached in IndexedDB)
  • Model weights (in browser Cache Storage when you load Gemma)

What leaves your browser

  • Fetches to naklitechie.github.io/parliamentwatch-data/ — static JSON + text files. No tracking.
  • Fetches to huggingface.co — model weights, only on first Gemma load.
  • If you use BYOK: requests directly to your provider. Your key isn't proxied through any other server.

What never happens

  • No analytics. No accounts. No telemetry. No server logs (no server).
  • The page is served as a static index.html from GitHub Pages.

Built on top of ParliamentWatch

The scraping logic, committee config, API map, and the original idea are all from ParliamentWatch by Pranay Kotasthane. The scraper.py, pdf_utils.py, and config.py in the data-mirror repo are vendored from his project. SansadLocal repackages it as a single HTML file with on-device AI; the data engine underneath is his.

Open-source pieces

  • Transformers.js v4 — runs Gemma in the browser via WebGPU
  • Gemma 4 (ONNX) — by Google, ONNX conversion by the HF community
  • pypdf — PDF text extraction in the daily mirror Action
  • Hosted free on GitHub Pages
